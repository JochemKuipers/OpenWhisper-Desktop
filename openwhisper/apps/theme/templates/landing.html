Why self-hosted private messaging?

Many teams need chat without surrendering message history to a third-party cloud. OpenWhisper keeps conversations on servers you operate, with a transparent Django codebase you can review in class or at work. You choose where PostgreSQL runs, how Redis is secured, and which users may register—ideal for labs, demos, and privacy-sensitive prototypes.

The web client talks to a REST API for history and uploads, while Django Channels delivers live updates over WebSockets. Friends, direct messages, and group chats share the same stack, so you learn one architecture instead of bolting on separate services. Start on localhost, then deploy behind your own reverse proxy when you are ready.

Whether you compare hosting options or document SEO for a school assignment, a single landing page explains the product while the authenticated app stays out of search indexes—a common pattern for SaaS and internal tools.

What is in the stack?

OpenWhisper pairs a Django 5 backend with PostgreSQL for durable message storage and Redis for channel layers, so WebSocket consumers stay responsive when several users chat at once. Session cookies protect the HTML shell, while SimpleJWT issues tokens for API calls and socket authentication—patterns you can document in class or reuse for mobile clients later. Profiles, friend requests, direct messages, and group chats all share the same REST and real-time pipeline, which keeps deployment, logging, and backups straightforward for labs and small teams.
